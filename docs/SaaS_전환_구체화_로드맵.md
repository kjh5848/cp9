# SaaS 전환 구체화 로드맵 (SaaS Transition Plan)

현재 개발 중인 CP9(Next.js 기반 웹 애플리케이션)을 실제 수익을 창출하는 구독형 **SaaS(Software as a Service)** 로 전환하기 위한 구체적인 단계별 실행 문서입니다.

## 1단계: 아키텍처 및 멀티테넌트(Multi-tenant) 인프라 구축
현재 단일 사용자 혹은 로컬 중심의 아키텍처를 다수의 사용자가 독립적으로 사용할 수 있는 환경으로 개편합니다.

*   **1.1. 데이터베이스 멀티테넌시 적용**
    *   **목표**: 사용자(Tenant)별 데이터 격리 및 보안 확보.
    *   **작업**: 모든 주요 테이블(캠페인, 스케줄, 포스팅 결과 등)에 `user_id` 또는 `workspace_id` 컬럼을 추가합니다.
    *   **보안**: Supabase(PostgreSQL)를 사용할 경우 **RLS(Row-Level Security)** 정책을 엄격하게 적용하여, 사용자가 자신의 데이터만 읽고 쓸 수 있도록 강제합니다.
*   **1.2. 비동기 백그라운드 작업 고도화**
    *   **목표**: 대규모 사용자의 AI 포스팅 작업을 안정적으로 동시 처리.
    *   **작업**: 현재의 단순 Cron API 호출 방식을 넘어, **메시지 큐(Message Queue)** 기반의 워커(Worker) 시스템(예: Inngest, Upstash QStash, Trigger.dev)으로 전환합니다. 이를 통해 재시도 로직, 실패 시 알림, 병렬 처리 병목 현상을 방지합니다.

## 2단계: 사용자 인증 및 권한 관리 (IAM)
*   **2.1. 소셜 로그인 및 이메일 인증**
    *   **작업**: Google OAuth, GitHub 등 주요 소셜 로그인과 마법사 링크(Magic Link) 기반의 이메일 인증을 도입합니다. (Supabase Auth 또는 NextAuth 활용)
*   **2.2. 사용자 역할 분리 (RBAC)**
    *   **작업**: `Admin`(시스템 관리자), `Pro User`(유료 구독자), `Free User`(무료 체험자) 등으로 권한(Role)을 분리하여 API 접근과 UI 노출을 제어합니다.

## 3단계: 수익화 및 결제 시스템 연동 (Billing)
SaaS의 핵심인 구독 모델(Subscription)을 구현합니다.

*   **3.1. 결제 게이트웨이(PG) 연동**
    *   **글로벌 대상**: Stripe 연동 (구독 관리, 결제 위젯, 영수증 발급 등 통합 제공).
    *   **국내 대상**: 포트원(Portone), 토스페이먼츠 등의 정기결제 API 연동.
*   **3.2. 요금제(Tier) 및 크레딧 시스템 설계**
    *   **Free Plan**: 월 3건 포스팅 제한, 기본 템플릿 제공.
    *   **Pro Plan**: 무제한 포스팅, 프리미엄 AI 페르소나, 커스텀 API 키 입력 허용 등.
    *   요금제에 따른 **기능 제한(Feature Gating)** 로직을 프론트엔드와 백엔드 미들웨어에 구현합니다.
*   **3.3. Webhook 처리**
    *   **작업**: 사용자의 결제 성공, 구독 취소, 카드 유효기간 만료 등의 이벤트를 결제사 Webhook으로 수신하여 데이터베이스의 `subscription_status`를 실시간으로 업데이트합니다.

## 4단계: 핵심 서비스 로직의 SaaS화
*   **4.1. API Key 관리 정책 (BYOK vs Managed)**
    *   **BYOK (Bring Your Own Key)**: 사용자가 자신의 OpenAI, 쿠팡 파트너스 API 키를 직접 입력하게 하여 SaaS 플랫폼의 토큰 비용 부담을 없앱니다. 이 경우 사용자의 API 키는 데이터베이스에 **AES-256 등급으로 암호화하여 저장**해야 합니다.
    *   **Managed**: SaaS 월 구독료에 AI 비용을 포함시키고, 플랫폼의 마스터 API 키를 사용합니다. 이 경우 사용자당 **월별 토큰 사용량(Usage Tracking)** 을 엄격하게 추적하고 제한(Rate Limit)해야 합니다.
*   **4.2. 외부 플랫폼(WordPress 등) 연동 계정 분리**
    *   사용자별로 각자의 WordPress 자격 증명(URL, App Password)을 안전하게 저장하고 관리할 수 있는 My Page 설정 화면을 고도화합니다.

## 5단계: 모니터링, 분석 및 고객 지원 (CS)
*   **5.1. 에러 및 로그 모니터링**
    *   **Sentry** 등을 도입하여 프론트엔드 런타임 에러와 백엔드 API 에러를 실시간으로 추적합니다.
*   **5.2. 사용자 행동 분석**
    *   **PostHog** 또는 Google Analytics를 부착하여 사용자가 어떤 기능을 가장 많이 쓰는지, 결제 전환율(Conversion Rate)의 병목이 어디인지 퍼널(Funnel)을 분석합니다.
*   **5.3. 관리자 대시보드 (Admin Panel)**
    *   전체 활성 사용자, 일일 생성된 포스팅 수, 서버 자원 상태 등을 한눈에 볼 수 있는 내부용 백오피스를 구축합니다.

## 6단계: 법적 고지 및 런칭 준비
*   **6.1. 이용약관(ToS) 및 개인정보처리방침 작성**
    *   사용자가 생성한 콘텐츠의 저작권 문제, 쿠팡 파트너스 활동에 대한 면책 조항, 개인정보 보관 기간 등을 명시한 문서를 준비하고 회원가입 시 동의를 받습니다.
*   **6.2. 랜딩 페이지 및 SEO 최적화**
    *   상품의 가치를 보여줄 수 있는 'Deep Tech' 감성의 랜딩 페이지를 구축하고 마케팅 플라이휠을 구동합니다.
