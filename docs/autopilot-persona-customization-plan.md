# 통합 페르소나 관리 & 커스텀 고도화 기획안

본 문서는 현재 시스템(오토파일럿, 키워드 추출, 아이템 기반 글 작성)에 하드코딩 또는 간단한 옵션으로 제공되던 페르소나 기능을, 사용자가 **자신만의 커스텀 페르소나를 기획/지정하여 모든 파이프라인에서 일관되게 적용**할 수 있도록 고도화하기 위한 단계별(Phase) 기획입니다.

## 🎯 목표 (Goal)
자동 발행 뿐만 아니라 **사용자가 직접 키워드를 고르고 수동으로 발행(수동 생성, 아이템 생성 등)하는 모든 과정**에서 천편일률적인 AI 톤앤매너를 탈피하고, 운영 중인 워드프레스 블로그의 정체성(예: '깐깐한 40대 주부', '데이터 기반의 얼리어답터' 등)에 완벽히 동화되게 합니다.

---

## 📅 고도화 페이즈 (Phases)

### Phase 1: 페르소나 템플릿 뱅크 (Persona Bank) DB 구축 및 전역 연동
어떤 생성 방식(자동/수동)이든 간에, 공통으로 바라볼 수 있는 페르소나 저장소가 필요합니다.

*   **독립된 페르소나 관리 기능 (CRUD)**
    *   `Persona` 모델(DB 테이블) 신설: `id`, `name`, `systemPrompt`, `toneDescription`, `negativePrompt(절대 쓰면 안 되는 표현)`
    *   어드민 메뉴에 [페르소나 뱅크] 탭 추가하여 사용자가 직접 프롬프트 관리 가능
*   **수동 발행 UI (아이템 리서치 / 키워드 추출 뷰) 연동**
    *   기존 검색/발행 세팅 창(Ex. WriteActionModal 등)에서 하드코딩된 'IT', '뷰티' 대신 **DB에서 불러온 커스텀 페르소나 목록**을 Dropdown으로 제공

### Phase 2: 오토파일럿(큐/자동 발행)への 페르소나 주입
사용자가 만든 '커스텀 페르소나'를 스케줄러(오토파일럿)가 읽어서 자동 발행할 수 있도록 연결합니다.

*   **DB 스키마 변경 (AutopilotQueue 확장)**
    *   `AutopilotQueue` 테이블에 `personaId` 컬럼 추가 (Phase 1에서 만든 Persona 테이블 참조)
*   **어드민 UI 구현**
    *   오토파일럿 새 키워드 등록 시, 큐에 넣을 때 페르소나 뱅크의 페르소나 중 하나를 지정하도록 UI 확장
*   **스케줄러 반영**
    *   `/api/cron/autopilot` 실행 시 `personaId`를 조회하여 해당 프롬프트를 SEO 파이프라인에 주입하여 실행

### Phase 3: 컨텍스트 및 스킬 자동 매칭 (지능형 단계)
페르소나는 단순한 말투 변환 이상이어야 합니다. 검색 의도(Intent)에 따라 AI가 페르소나 내의 다른 '스킬'을 발현해야 합니다.

*   **카테고리별 동적 매칭 엔진 구현**
    *   키워드가 "노트북"일 경우 -> [IT 전문가] 페르소나 자동 차용 + 스펙 비교표(Table) 능력 활성화
    *   키워드가 "수분크림"일 경우 -> [뷰티 큐레이터] 페르소나 자동 차용 + 성분 분석표 능력 활성화
*   **효과**: 사용자가 키워드만 던져두어도, 시스템이 알아서 상품군에 맞는 자아를 장착하고 글을 기획하게 됩니다.
