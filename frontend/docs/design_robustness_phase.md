# 워드프레스 디자인 호환성 개선 가이드 (Design Robustness Phase)

## 📌 개요
CP9 파이프라인에서 생성한 SEO 아티클이 워드프레스(WordPress)로 전송될 때 기본 제공 테마 외('클린 블루' 등)에서 레이아웃이 깨지거나, 버튼이 파란색 알약 형태의 빈 박스로 중복 생성되는 현상을 고도화 및 해결한 페이즈입니다.

## 🚨 문제 원인: 워드프레스 `wpautop` 필터
워드프레스는 코어 로직으로 글 본문 저장 또는 출력 시 `wpautop`라는 필터를 거칩니다. 이 필터는 다음과 같은 매우 강력(때로는 공격적)인 HTML 변형을 일으킵니다.

1. **줄바꿈 자동 인식 (`<br>` 및 `<p>` 주입)**
   - HTML 텍스트 블록 안의 단일 줄바꿈(`\n`)을 강제로 `<br>` 태그로 치환합니다.
   - 두 번 연속된 줄바꿈(`\n\n`)을 감지하면 강제로 태그를 단락(`<p>`)으로 분리합니다.
   - *결과:* Flex 구조나 중앙 정렬(`text-align: center`) 디자인에서 예고 없는 여백 및 요소 틀어짐이 생겨 디자인 테마가 깨집니다.

2. **블록 태그(Block Tag) 안쪽 규칙 위반 시 강제 쪼개기**
   - 워드프레스는 `<a>`(인라인 요소) 안에 `<div>`나 `<p>` 같은 블록(Block) 요소가 들어가는 것을 불법 마크업으로 취급하여 태그를 강제로 닫고 다시 엽니다.
   - 예: `<a href="..."><div>아이콘 + 텍스트</div></a>`
   - 변환 후: `<a href="..."></a><div>아이콘 + 텍스트</div><a href="..."></a>`
   - *결과:* 정작 링크가 걸린 `<a>` 태그는 텅 빈 상태로 CSS 렌더링이 적용되어 (패딩과 배경색만 남은) '빈 공간의 파란색 버튼'들이 화면에 덩그러니 표출되는 버그를 유발합니다.

## 🛠 해결 방안 (고도화 내역)

### 1. `<a>` 태그 내부 구조 평탄화 (인라인 변환)
`seo-cta-builder.ts` (`buildCtaHtmlSnippet` 등) 내부의 CTA 버튼 마크업에서:
- 기존: `<a><div><svg></svg><span>텍스트</span></div></a>`
- **변경**: `<a><svg></svg><span>텍스트</span></a>`
`<div>` 태그를 제거하고, `<a>` 태그 자체에 `display: inline-flex` 등 플렉스 정렬 속성을 직접 부여하여 워드프레스가 요소를 강제로 분해하지 않도록 마크업을 단일화했습니다.

### 2. 백틱(Template Literal) 내 줄바꿈 완전 제거
`cta-html-builders.ts`, `cta-constants.ts` 전반에서:
- 기존: 가독성을 위해 `<div ...>\n  <p>...\n  <a>...</a>\n</div>` 형태의 템플릿 사용
- **변경**: `<div ...><p>...</p><a ...>...</a></div>` (한 줄 코딩 적용)
`buildPriceBlock`, `buildMidContentCta`, `buildDynamicCta`, `buildCtaBlockHtml` 등의 모든 리턴 문자열을 평탄화(Flatten)하여 `wpautop` 필터가 `<br>`을 집어넣을 틈을 주지 않도록 개선했습니다.

## 💡 향후 규칙 (Rule)
앞으로 새로운 프론트엔드/백엔드 디자인 컴포넌트나 CTA 템플릿을 추가할 경우 다음 규칙을 무조건 준수해야 합니다.
1. **문자열 엔터 공백 금지:** TS 파일 내에서 워드프레스로 전해질 본문 HTML (템플릿 문자열 등) 내부 태그 사이에 실제 키보드 엔터(`\n`)를 사용하지 마십시오.
2. **`<a>` 태그 내 블록 요소 사용 금지:** Anchor 태그 내부에는 `<div>`나 `<p>` 대신 `<span>` 과 `<svg>` 같은 인라인 요소만 배치하십시오. 정렬이 필요하다면 `<a>` 태그 자체에 `inline-block`이나 `inline-flex` 스타일을 사용해야 합니다.
