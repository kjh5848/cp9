# Project Debugger (프로젝트 분석 및 디버거)

이 스킬은 프로젝트의 버그, 구조적 문제, 반복되는 에러를 찾아내고 근본적인 개선을 수행하기 위한 가이드입니다.

## 핵심 행동 수칙 (Core Directives)

이 스킬이 트리거되면, **코드를 바로 수정하거나 추측성 답변을 하기 전에 반드시 아래 단계를 강제로 수행**해야 합니다.

1. **프로젝트 환경 및 룰 파악 (강제 실행)**
   - 프로젝트 내에 존재하는 사용자 정의 규칙(User Rules), `.agents/workflows/`, `.agents/skills/` 폴더 내의 관련 가이드를 먼저 읽으세요 (`view_file`, `list_dir` 활용).
   - 프로젝트의 아키텍처(예: FSD 아키텍처) 및 스타일 가이드라인을 먼저 파악하여, 기존 코드의 문맥을 부수지 않도록 주의합니다.
   
2. **시스템적 원인 분석 (Systematic Debugging)**
   - 에러 로그나 터미널 출력(`command_status`, `read_terminal`)을 확인합니다.
   - 단편적인 오류 수정(예: null check 추가)에 그치지 말고, 데이터 흐름과 상태 관리의 **근본적인 병목(Root Cause)**을 파악하세요.
   - 필요하다면 사용 중인 의존성 스킬(`systematic-debugging`, `debugging-strategies`)의 아이디어를 차용하여 여러 가설을 세우고 하나씩 검증합니다.

3. **가설 검증 및 개선안 제시**
   - 사용자에게 문제의 원인(Why)과 해결 가설(Hypothesis)을 명확하게 설명합니다.
   - 임시 조치가 아닌 구조적 개선을 제안하고, 동의를 얻으면 코드를 수정합니다.

## 디버깅 접근 전략

- **Frontend (React/Next.js)**: 상태 동기화 문제(useEffect 남용), 불필요한 리렌더링, 컴포넌트 마운트 시점 에러(Hydration), 클라이언트/서버 컴포넌트 경계(`"use client"`) 오류를 최우선 의심하세요.
- **Backend (API/DB)**: 네트워크 페이로드 검증 누락, 데이터베이스 쿼리 교착상태(Lock/Timeout), 프라미스(Promise) 비동기 처리 누락을 확인하세요.
- **AI Pipeline (LLM)**: 프롬프트 주입 오류, 파싱 포맷 불일치(JSON 등), 타임아웃, 예기치 않은 토큰 생성(이모지 등)을 확인하세요.

## 제약 사항

- **절대 추측으로 코드를 작성하지 마세요.** 반드시 `grep_search`나 `view_file`로 실제 구현 코드를 확인한 후 수정해야 합니다.
- 수정한 후에는 프로젝트의 터미널 명령(예: `npm run build`, `npm run dev`) 상태 로그를 관찰하여 추가적인 연쇄 오류가 발생하지 않았는지 확인합니다.
