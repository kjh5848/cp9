# SEO 파이프라인 자동화 워크플로우 (`/seo-pipeline`)

이 워크플로우는 사용자가 지정한 '주제'와 '글자 수' 조건에 맞춰, 외부 최신 데이터를 검색하고 블로그 템플릿에 맞게 양질의 SEO 포스팅 초안을 생성하는 전체 과정을 가이드합니다.

## 작동 원리 및 권장 프로세스

에이전트(Claude 등)는 아래의 3단계 파이프라인 지침을 순차적으로 수행해야 합니다.

### 1단계: Perplexity를 통한 최신 스펙/가격 데이터 수집 (Data Fetching)

- **트리거**: 사용자가 제시한 주제(예: "제네바 스피커 XL 렌탈")에 대해 최신 정보의 공백이 예상될 경우.
- **행동 지침**: 
  - `test_perplexity.ts` 등의 로컬 API 스크립트 도구를 실행하거나, 내장된 웹 검색(또는 MCP 도구)을 활용하여 타겟 제품의 실사용자 단점, 최신 가격, 스펙, 벤치마크 데이터를 수집합니다.
  - 검색어 예시: `"제네바 스피커 XL 2025년 기준 실사용 단점 비교 및 최저가 리뷰"`
  - 환각(거짓 정보)을 방지하기 위해 이 단계의 결과를 반드시 기반 자료로 확보합니다.

### 2단계: 스킬(seo-article-writer) 기반 블로그 본문 작성 (Content Generation)

- **트리거**: 1단계에서 팩트 데이터가 준비되었을 경우.
- **행동 지침**: 
  - `@[/seo-article-writer]` 스킬의 `SKILL.md` 및 `blog_prompt_template.md` 가이드를 불러옵니다.
  - 수집한 최신 데이터를 재료로 삼아, 사용자가 요구한 **글자 수(기본: 최소 1,500 ~ 2,500자)** 에 맞춰 본문을 생성합니다.
  - 다음 요소가 반드시 포함되어야 합니다:
    1. 흥미롭고 공감대를 형성하는 오프닝
    2. 필수 스펙 비교표 및 장단점 분석 (1단계의 팩트 데이터 활용)
    3. 추천 대상 및 가치 입증
    4. 체류 시간을 늘리기 위한 시각적 포맷팅 (소제목 `##`, 불릿 포인트 `-` 활용)
    5. '마스터 큐레이터 H' 페르소나 적용

### 3단계: 파일 저장 및 썸네일/이미지 기획 (Finalizing & Visualizer)

- **행동 지침**: 
  - 작성된 기사 결과물을 `.md` 파일 형식으로 저장합니다. (예: `기사작성결과_[주제명].md`)
  - 저장 시 파일 내부의 적절한 문단 사이에 DALL-E 3나 Midjourney 프롬프트로 활용할 수 있는 **[시각화 프롬프트 제안]** 을 2~3개 삽입해 줍니다.
  - 예시: `[이미지 제안: 따뜻한 조명이 켜진 모던한 거실 인테리어 중심에 놓인 제네바 스피커 XL 월넛 색상 이미지]`

## 실행 예시

사용자가 워크플로우를 호출하면, 에이전트는 다음과 같이 피드백하며 작업을 시작합니다.

1. **사용자 입력**: "`/seo-pipeline` [주제: 제네바 스피커 XL 렌탈] [글자수: 2000자 이상]"
2. **에이전트 응답 및 실행**:
   - "네, SEO 파이프라인을 가동합니다. 1단계: Perplexity를 통해 '제네바 스피커 XL'의 최신 렌탈 트렌드와 사용자 리뷰를 수집하겠습니다..." (검색 진행)
   - "2단계: 수집된 데이터를 바탕으로 `seo-article-writer` 가이드에 따라 2000자 이상의 초안 작성을 시작합니다..." (기사 작성)
   - "3단계: 작성 완료되었습니다. 작업 디렉토리에 `geneva_xl_rental_review.md` 파일로 저장해 두었으며, 본문 내에 인테리어 이미지 프롬프트 제안을 포함했습니다."

이 워크플로우를 통해 나중에 백엔드 서버(Node.js/Next.js)에서 실행할 완벽한 LLM 파이프라인 로직의 뼈대를 테스트하고 정립할 수 있습니다.
