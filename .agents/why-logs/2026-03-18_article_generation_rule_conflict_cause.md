# 워크플로우와 글쓰기 생성 간의 프롬프트 충돌 문제

## 🚨 발생 문제
- LLM이 SEO 블로그 파이프라인(`html-phase.ts`, `article-phase.ts`)을 통해 글을 작성할 때, "이모지 사용 금지" 및 "매크로만 포함된 정형화된 CTA" 지침을 따르지 않고 이모지가 포함된 H2 제목과 비정형적인 CTA 문구를 생성하는 문제가 발생함.
- `html-phase.ts` 백엔드 로직이 이모지(`emojiRegex`)를 강제로 삭제 처리하여, `## 💡 [상품명]`이 `##  [상품명]` 등으로 변환되며 공백/포맷팅 오류가 발생.
- CTA 또한 백엔드는 `[[[CTA_BUTTON:URL]]]` 매크로만을 탐색하여 컴포넌트를 주입하려 했으나, LLM이 매크로 없이 일반 추천 문구를 써버려 CTA 주입에 실패하거나 레이아웃이 깨짐.

## 🔍 원인 분석
- **분석 관점**: Rules (글로벌 룰 및 스킬 지침) vs Prompt Template (실제 주입되는 템플릿)의 상충
- **근본 원인**: 
  1. `seo-article-writer/SKILL.md`와 백엔드 API 시스템 프롬프트(`article-phase.ts`)는 명시적으로 "절대 이모지/아이콘 사용 금지"를 지시하고 있으나, 정작 기반이 되는 `blog_prompt_template.md` 파일 내의 `[본문 구성 제안]` 섹션에는 `## 💡`, `## 📊`, `✅치명적 단점`, `🏆장점` 등 이모지가 명시된 가이드라인이 포함되어 있어 프롬프트 역설(Paradox)이 발생함.
  2. 시스템 프롬프트는 "CTA 삽입 시 오직 `[[[CTA_BUTTON:url]]]` 매크로만 작성하라"고 지시하지만, `blog_prompt_template.md`는 "글 하단에 어울리는 다른 하이엔드 아이템 리뷰로 연결되는 CTA 문구를 작성하십시오."라고 텍스트 작성을 장려하여, LLM이 문구를 임의로 추가하게 됨.

## 💡 해결 및 재발 방지 방안
- `blog_prompt_template.md` 내의 모든 이모지 예시(`💡`, `📊`, `✅`, `🏆`, `💰`, `🎯`)를 제거하고 일반 괄호 형태(`[단점]`, `[장점]`)로 치환함.
- 템플릿의 CTA 작성 지침을 수정하여 "오직 `[[[CTA_BUTTON:URL]]]` 매크로만 작성하십시오. 다른 HTML이나 부가 텍스트는 절대 포함하지 마십시오."라고 규칙을 통일함.
- **재발 방지 전략 (Best Practice)**: 
  - 백엔드 프롬프트(`article-phase.ts`)의 엄격한 시스템 룰과, 사용자 혹은 스킬에서 주입하는 템플릿(`blog_prompt_template.md`)의 예시가 충돌하지 않도록 모순점을 주기적으로 검증해야 함.
  - "하지 마라(Negative)"는 지시보다 "A 형태로만 100% 작성하라(Positive)"는 포맷을 예시와 동일하게 맞춰주는 것이 LLM 파이프라인의 환각(Hallucination) 방지에 훨씬 효과적임.
