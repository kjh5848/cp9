# CTA 렌더링 실패 및 프롬프트 금지어 규칙 무시 문제

## 🚨 발생 문제
- 마크다운 파싱 결과에서 `[[CTA_BUTTON:URL...]]` 버튼 매크로가 HTML 컴포넌트로 치환되지 않고 단순 텍스트로 노출됨.
- LLM 소제목 생성 시 금지된 단어인 "결론" 및 쌍점(콜론, `:`)을 무분별하게 사용하여 자연스러운 블로그 포맷을 해침.

## 🔍 원인 분석
- **분석 관점**: Code Quality (정규식 엄격성 오류), Rules (프롬프트 내 명시적 규칙 부재)
- **근본 원인**: 
  1. **CTA 렌더링 측면**: `html-phase.ts` 내부의 CTA 변환 정규식이 대괄호 3개(`[[[ ]]]`)인 경우만 매칭하도록 좁게 설정되어 있어, LLM이 문맥상 대괄호를 2개만 출력했을 때 치환 로직이 실패함.
  2. **프롬프트 측면**: 블로그 정보글을 생성하는 LLM의 본능상 구조화된 데이터 표현을 위해 `결론:`, `특징:` 등의 포맷을 자연스럽게 사용하려 하는데, 기존 `article-phase.ts`의 지시사항에는 이를 명시적으로 차단하는 강력한 **"네거티브 프롬프트"** 가 부재했음.

## 💡 해결 및 재발 방지 방안
- `html-phase.ts` 정규식을 `\[{2,3}CTA_BUTTON...` 형태로 개선하여 대괄호 2개 혹은 3개 생성을 모두 관대하게 수용하고 컴포넌트로 정상 치환하도록 수정함.
- `article-phase.ts` 내 모든 템플릿(단일, 비교, 큐레이션, 키워드 모드)의 [공통 절대 규칙]과 [강제 지침] 상단에 **'결론'** 이라는 단어 및 **'쌍점(:)'** 기호 사용을 절대 금지하는 핵심 지침을 명시함.
- **재발 방지 전략**: 이후 프롬프트 기획 시 특정 기호나 단어가 LLM의 기본 본능(예: 구조화, 요약)과 겹칠 수 있음을 먼저 인지하고, 금지어 로직 추가 시에는 해당 텍스트나 기호를 원천 차단하는 독립된 조항을 확실하게 심어두는 패턴을 따릅니다.
